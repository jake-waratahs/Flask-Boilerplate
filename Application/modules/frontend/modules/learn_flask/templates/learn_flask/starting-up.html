Starting Up

By this stage, you've seen some very basic flask apps. You should be partially familiar with running a flask app, since you did that in the earlier section, using python myapp.py. We will now move into starting up using Flask-Boilerplate

Flask-Boilerplate relies on one global package installed on your computer: flask-boilerplate-buildutils. This package manages things such as the virtual environment and how configuration gets chosen for your app depending where you are developing.

Firstly, acquire a copy of this boilerplate, either by downloading it from github as a zip or cloning it. Once you have it, cd into the root directory.

From here, execute source .env in your terminal. This will install flask-boilerplate-buildutils, set up a virtual environment and install all the requirements, including config specific requirements (provided you have the environment variable FLASK_CONFIG set). BONUS: If you have autoenv installed, the script would have already run without you typing anything.

Everything should now be installed, and you should be ready to develop. Start the development server by typing:

python run.py
 * Running on http://127.0.0.1:8000/ (Press CTRL+C to quit)
 * Restarting with stat

Pull up your web browser and you should see this exact website. Awesome!
