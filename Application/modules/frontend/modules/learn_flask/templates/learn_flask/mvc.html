MVC - Models, Views, Controllers

Flask applications can be as small as a single page, or as large as an e-commerce website. Most of the time you'll be developing an application which requires a database (Models), multiple pages to present to the user (Views), and ways for the user to interface with the app (Controllers).

Most tutorials which cover Flask don't cover "Going large", which is a very big problem. Generally, it is wise to split up the specific parts of your application into smaller more broken down parts.

The Flask library makes heavy use for the term "Template". For this reason, we will refer to MVC "Views" as Templates.
