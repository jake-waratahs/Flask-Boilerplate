WSGI vs CGI

Who has programmed in PHP?

In most cases, when developing a PHP application, you use CGI to deploy and develop it. CGI is the process of executing the requested file, and returning the stdout output to the browser. Quite simple in principle, however difficult to get set up and annoying to develop with. Every time you wish to develop, you need to configure apache/nginx and setup fastcgi on your local machine.

WSGI was originally designed for the Python language, and stands for Web Server Gateway Interface. Originally, python web applications required serving via CGI, in a very similar manner to PHP websites.

Flask WebApps run as their own contained instances. Flask includes a builtin WSGI server (for development only). When deploying, a WSGI server, such as Gunicorn or meinheld should be used. A WSGI server provides a an interface between your Python instance and HTTP requests.
