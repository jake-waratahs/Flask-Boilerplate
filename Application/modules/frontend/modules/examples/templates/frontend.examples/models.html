Models

This boilerplate encourages users to use an ORM with their Flask app. This page details the usage of Flask-SQLAlchemy with your Flask app. It must first be noted, however, instead of using Flask-SQLAlchemy's model pattern, we instead use raw SQLAlchemy models. This allows usage of models across multiple apps, and non flask-apps.

Locating the Models

All models for this flask app are contained within ./libs/models/models. This is a pip/setuptools installable package, which means it is globally accessible (within your virtual environment). It can be imported at any time with a single line: import models

It is highly recommended that you stucture your models into subdirectories such as models.core, models.auth, etc. This helps when going big.
