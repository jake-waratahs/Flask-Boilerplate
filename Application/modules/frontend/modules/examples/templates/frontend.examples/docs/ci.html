Continuous Integration

Deploying a Flask webapp to a testing server is a reasonably simple process.

Configure a CI build script on your CI server to do the following:

export FLASK_CONFIG=CI &&
source .env &&
python3 run.py test

This is assuming the current directory is the root of your Flask app where run.py is located.

Writing Tests

Inspect the folder ./ApplicationTests to get an understanding on how tests are structured. It is recommended that you split tests cases into multiple files to allow easier access when a specific test needs to be worked on.
