Develop using MySQL

It's often quite nice to develop locally using a MySQL database, but it's not always very easy to do. It generally requires setting up a username and a password for the specific use.

In a development environment, instead of setting up app specific user accounts and databases, a general "dev" user should be created (with local only access). It should only have privileges to databases prefixed with "dev".

Setup MySQL Account for Development

Open a MySQL console using a user that has root privileges

mysql -u  -p

-- Create the user account
CREATE USER 'dev'@'localhost';
-- Grant Privs for the dev account
GRANT ALL PRIVILEGES ON `dev%`.* TO 'dev'@'localhost';
-- Exit Mysql
EXIT;

Set Environment Variables

To tell Flask to use MySQL for all your projects, set the FLASK_CONFIG environment variable. If you wish to only use MySQL in a single application, simply invoke it using the -c MySQLStd argument.

echo "export FLASK_CONFIG=MySQLStd" >> ~/.bashrc
echo "export FLASK_CONFIG=MySQLStd" >> ~/.zshrc

Once you've set those environment variables, you'll be required to re-open your shell for them to be loaded.
