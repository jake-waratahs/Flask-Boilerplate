Helpers

Strictly speaking, the boilerplate does not come with any helpers. All helpers are included from the flask-boilerplate-utils package. This package brings the following helpers:

Run Commands

See Running for more info about run commands

Jinja Filters

  • timesince: Return a string representing a time since a particular date, ie, 3 days ago, etc.
  • local_date: Returns a babel formatted local date (requires BABEL_ENABLED=True config option).
  • percent_escape: Percent escape a string using percentage symbols (URL Encode)
  • local_date_time: Returns a babel formatted local date and time (requires BABEL_ENABLED=True config option).

Jinja Globals

  • {{ '{{ csrf_setup }}' }}: Returns a CSRF script tag which patches all AJAX requests to send a CSRF token before sending.
  • {{ '{{ render_field }}' }}: Render a WTForms form field object into HTML code.

Forms

  • RequireItemAsMember: A WTForms validator to check if the form data contains a particular item.
  • ValidFileFormat: Validate a fileupload to ensure the uploaded file is in the file upload's allowed types
  • StopIfEqualTo: Stop validation chain if the field value is equal to the getter's return value.
  • Unique: Validator to ensure data doesn't already exist in the database.
  • OptionFileField: Ignore validation if a file is not provided for upload.
  • RequireAny: Require one or more of the selected fields to have data.
  • empty_string_none: A WTForms filter to turn an empty string into a None
  • TimezoneDateTimeField: A timezone aware field for WTForms which understands javascript getTimezoneOffset()

Overrides

  • NestableBlueprint: Nest blueprints within other blueprints
  • FlaskView: Configure Flask-Classy to ignore specific kwargs/args when converting them to an endpoint mapping. Allows blueprints to have dynamic path arguments and allow endpoints to catch them in its args/kwargs.

All of the helpers listed here are documented in the readthedocs documentation for flask-boilerplate-utils over here
